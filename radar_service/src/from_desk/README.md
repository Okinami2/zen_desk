\# 🛰️ 智能桌面人员状态监测系统 (Intelligent Desktop Monitor)



基于 HLK-LD2420 24GHz 毫米波雷达与 C 语言开发的高精度、抗干扰人员在座状态监测系统。本项目专为大学自动化专业大作业设计，旨在通过高级数字信号处理算法与状态机逻辑，解决传统红外传感器无法检测静态人体、以及雷达传感器在极近场/远场存在的物理局限性。



\## ✨ 核心工程痛点与算法创新



本项目摒弃了雷达自带的粗糙“有人/无人”高低电平判定，全面接管雷达底层 16 个距离门的原始能量数据，并在 C 语言应用层实现了以下三大核心算法：



\* \*\*方差双擎驱动 (Variance Dual-Engine)\*\*

&#x20;   \* \*\*痛点\*\*：在 0\~70cm 的极近场，天线自耦和桌面静态反射导致底层能量长期处于满载爆表状态（均值失效）。

&#x20;   \* \*\*解决\*\*：引入 10 帧（1秒）滑动窗口，实时提取空间能量波动的\*\*方差\*\*。利用“方差突破死水状态”精准捕捉微弱呼吸与打字动作。

\* \*\*空间分治策略 (Spatial Divide \& Conquer)\*\*

&#x20;   \* \*\*痛点\*\*：远距离环境存在强烈的“多径效应”（墙壁折射），导致远处的门出现与人体波动一致的“高方差幽灵”。

&#x20;   \* \*\*解决\*\*：近场（0-1门）旁路均值，100% 依赖高灵敏方差；远场（2-15门）重拾能量均值门限，以绝对能量大小过滤多径幽灵干扰。

\* \*\*漏桶容错有限状态机 (Leaky Bucket FSM)\*\*

&#x20;   

&#x20;   \* \*\*痛点\*\*：入座时偶有停顿会导致积分清零；而在座时深度思考或屏息会导致传感器短时丢帧。

&#x20;   \* \*\*解决\*\*：采用极度非对称的时间常数。入座采用“漏桶算法”（轻度扣分制，容忍 1-2 秒发呆）；离座采用 30 秒长时延兜底，构建巨大的“防抖死区”，彻底杜绝“人在灯灭”的尴尬。



\## 📂 工程目录结构



严格遵循 C 语言模块化编程规范（兼容 C89/C90 标准）：



```text

├── main\_monitor.c      # 【主程序】高级监控核心，包含 FSM、方差计算与空间分治算法

├── main\_observer.c     # 【辅助工具】全能量观测器，用于前期标定环境空载底噪

├── radar\_protocol.c    # 【协议层】海凌科雷达十六进制数据帧解析与软控指令组装

├── radar\_protocol.h    # 【协议层】数据结构体与接口定义

├── serial\_utils.c      # 【驱动层】基于 Windows API 封装的无阻塞串口读写

└── serial\_utils.h      # 【驱动层】串口驱动接口定义

```



\## 🛠️ 编译与运行环境



\* \*\*操作系统\*\*：Windows

\* \*\*编译器\*\*：GCC (MinGW)

\* \*\*硬件要求\*\*：HLK-LD2420 毫米波雷达模块、USB转TTL串口模块



\### 🚀 编译指令



请在 VS Code 终端（确保已安装 gcc）中执行以下命令：



\*\*1. 编译高级监控主程序（日常使用）：\*\*

```bash

gcc main\_monitor.c serial\_utils.c radar\_protocol.c -o monitor.exe

```



\*\*2. 编译全能量观测工具（底噪标定用）：\*\*

```bash

gcc main\_observer.c serial\_utils.c radar\_protocol.c -o observer.exe

```



> \*\*⚠️ 注意事项\*\*：编译前请务必打开 `main\_monitor.c` 和 `main\_observer.c`，将宏定义 `#define PORT\_NAME "\\\\\\\\.\\\\COM11"` 修改为你电脑设备管理器中实际的串口号！



\## 🎮 系统操作指南



运行 `.\\monitor.exe` 后，系统将进入无阻塞监听模式，你可以通过键盘随时干预雷达运行状态：



| 快捷键 | 功能说明 | 备注 |

| :--- | :--- | :--- |

| \*\*`0` \~ `9`\*\* | 切换监视 0 \~ 9 号距离门 | 切换时自动清空历史数据，防止状态污染 |

| \*\*`a` \~ `f`\*\* | 切换监视 10 \~ 15 号距离门 | `a` 对应 10，以此类推 |

| \*\*`R` / `r`\*\* | 硬件级热重启 | 下发复位指令并暴力清洗底层驻留脏数据 |

| \*\*`Q` / `q`\*\* | 雷达待机（挂起） | 下发配置指令暂停射频发射，降低功耗 |

| \*\*`S` / `s`\*\* | 雷达唤醒 | 退出配置模式，恢复数据上报 |



\## 📊 状态跃迁说明



系统具有三级人员状态，并会在状态发生越阶时输出极具辨识度的 `\[EVENT]` 弹窗拦截日志：



1\.  \*\*没人 (AWAY)\*\*：目标彻底消失 30 秒以上。

2\.  \*\*微动 (NORMAL)\*\*：人员已入座，保持日常呼吸、打字、阅读等微小动作。

3\.  \*\*乱动 (FIDGET)\*\*：人员在座，且正发生换座位、伸懒腰等大幅度截面积变化的动作。
